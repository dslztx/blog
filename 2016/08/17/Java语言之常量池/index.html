Java语言之常量池

发表于 2016-08-17 专题于 语言 Disqus：

本文介绍Java语言中经常提及的“字符串常量池”，“包装器类常量池”和“运行时常量池”。不同JDK版本，采用的机制和实现有可能不同，本文选定JDK7版本。

一、字符串常量池

1.1、概念

字符串常量池，英文名称为“String Literal Pool”或者“String Constant Pool”，处于Java堆中，是相应的复用String实例对象机制中逻辑上划分得到的一块内存区域。
具体为：
Java堆中现有一个String实例对象aa0，它的引用值为ab0，欲新建一个String实例对象引用变量ac0，它引用到的String实例对象包含的字符串跟aa0包含的字符串一致，有两种方案：1）在Java堆中，新建一个String实例对象aa1，aa1包含的字符串跟aa0包含的字符串一致，它的引用值为ab1，令ac0=ab1；2）Java堆中维护一个String实例对象引用值集合，其中存储一些String实例对象的引用值，这里先存储aa0的引用值ab0，现在可以遍历查询该集合，获得ab0，令ac0=ab0，达到目的。
以上两个方案中，方案2相较于方案1能够复用String实例对象，从而节省Java堆内存。方案2中涉及到的“存储String实例对象引用值的集合”和“所有集合中引用值引用到的String实例对象”构成逻辑上的“字符串常量池”内存区域。示意图如图1所示。

图1

1.2、可行性分析

Java语言中，String实例对象不可修改，因此，“多个String实例对象引用变量引用同一个String实例对象”在节省Java堆内存的情况下，并不会带来任何问题。

1.3、实际运行过程

在实际运行过程中，有以下4个关键概念点。

  1. 上述提及的“存储String实例对象引用值的集合”实际上是一个类似于Map<Integer,List<String>>的对象（实际中是一个StringTable类型的对象）。它的key为hashCode值，它的value为具有对应的hashCode值的一系列String实例对象的引用值集合
  2. 调用String类的“intern()”本地方法，是唯一的与“字符串常量池”内存区域进行作用的途径
  3. 只有String实例对象才能调用String类的“intern()”本地方法，因此，使用“字符串常量池”机制，虽然最终可以复用String实例对象，但是中间过程中生成“临时的String实例对象”在所难免，这些“临时的String实例对象”最后由于“引用不可达”而被垃圾回收
  4. 对每一个字符串字面量（如“hello”，“world”等以直接形式给出的字符串），JVM会创建一个String实例对象，然后自动隐式调用其上的“intern()”方法

二、包装器类常量池

同样基于节省内存的考虑，“Boolean，Character，Byte，Short，Integer，Long”这几个基本类型的包装器类提供实现了常量池机制，具体可查看相应类的“valueOf()”方法。

另外，对于“Boolean，Character，Byte，Short，Integer，Long”这些包装器类相应的基本类型的字面量值，Java采用了跟String一样的机制：运行时，都会自动隐式调用一个类似于“intern()”方法的方法，此时，这个类似于“intern()”方法的方法跟相应类的常量池产生联系。由这些包装器类的源代码可知，常量池只存储了一定范围内的值，这也就限制了能够使用常量池机制的字面量值的范围。

三、运行时常量池

运行时常量池，英文名称为“Runtime Constant Pool”。Class文件中有一部分信息是常量池，它的英文名称为“Constant Pool Table”，在运行时，它其中的内容会被载入到相应的运行时常量池中。由以上可知，在运行时，一个Class文件的常量池对应一个运行时常量池。

四、三者的区分

字符串常量池，包装器类常量池和运行时常量池区分如表1。

表1

常量池名称 所处内存区域 存储内容 描述
字符串常量池 Java堆 “存储String实例对象引用值的集合”和“所有集合中引用值引用到的String实例对象” 复用String实例对象，节省内存
包装器类常量池 Java堆和方法区 除了Boolean类之外，存储数组实例对象引用值的静态变量处于方法区中，数组实例对象处于Java堆中，包装器类实例对象处于Java堆中；Boolean类中，TRUE和FALSE这两个静态变量存储于方法区中 复用包装器类实例对象，节省内存
运行时常量池 方法区 存储运行时载入的Class文件中的常量池内容 一个Class文件的常量池对应一个运行时常量池

五、JDK6中的字符串常量池机制和实现

JDK6中的“字符串常量池机制和实现”跟JDK7中的“字符串常量池机制和实现”相比有很大不同。

  • 在JDK7的“字符串常量池机制和实现”中，“字符串常量池”内存区域处于Java堆中；而在JDK6的“字符串常量池机制和实现”中，“字符串常量池”内存区域处于方法区中
  • JDK7中的“intern()”方法的实现逻辑是：如果StringTable对象中没有一个引用值指向的String实例对象与当前String实例对象的内容相同，那么在StringTable对象中新建一个引用值变量，存储当前String实例对象的引用值，否则返回已存在的引用值；而JDK6中的“intern()”方法的实现逻辑是：如果StringTable对象中没有一个引用值指向的String实例对象（如果存在，处于“字符串常量池”中）与当前String实例对象（处于Java堆中）的内容相同，那么在“字符串常量池”中复制新建一个String实例对象和一个引用值变量，该String实例对象的内容（处于“字符串常量池”中）与当前String实例对象（处于Java堆中）的内容相同，该引用值变量存储该复制新建String实例对象的引用值，否则返回已存在的引用值
